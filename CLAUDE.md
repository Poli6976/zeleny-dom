# Security Rules — ОБЯЗАТЕЛЬНО СОБЛЮДАТЬ

Эти правила действуют для проекта **«Расти Дома»** (`D:\Рабочий стол\zeleny-dom`,
русскоязычный сайт на Astro, задеплоен на GitHub Pages → https://rasti-doma.ru)
и любого добавляемого кода (скрипты, автономный лунный календарь `public/lunagarden.html`,
Python-утилиты). Те же правила действуют для второго проекта Leafwise.

> Активный проект пользователя — этот («Расти Дома»). Leafwise — второстепенный, в отдельной
> папке `D:\Рабочий стол\Pr пасивного дохода`. Не путать папки/проекты.

## Секреты
- НИКОГДА не хардкодить API-ключи, пароли, токены в код.
- Всегда использовать переменные окружения: `process.env` (Node) или
  `python-dotenv` / `os.environ` (Python).
- Всегда добавлять `.env` в `.gitignore` (уже сделано).
- Секреты для CI хранить в GitHub → Settings → Secrets, не в workflow-файлах
  (например `ANTHROPIC_API_KEY` для генерации статей).

## База данных
- Только параметризованные запросы (prepared statements / плейсхолдеры `?`).
  НИКАКОГО string-interpolation или шаблонных строк в SQL.
- Никаких деталей об ошибках БД/стека в HTTP-ответах — клиенту только
  обобщённое сообщение, подробности логировать на сервере.

## Аутентификация
- Вся логика прав (авторизация) — только на сервере, НИКОГДА в клиентском JS.
- Хеширование паролей: bcrypt или argon2 (никогда не хранить в открытом виде,
  никогда не логировать пароли).
- Сессионные cookie: `httpOnly`, `secure` (в проде), `sameSite=strict`;
  регенерировать сессию после логина (защита от session fixation).
- Единообразные ответы на ошибки логина/регистрации (защита от user enumeration).
- Ограничение частоты (rate limiting) на эндпоинты входа/регистрации.
- CSRF-защита на все изменяющие состояние запросы.

## Зависимости
- Использовать только популярные, проверенные пакеты.
- Указывать точные версии в `package.json` / `requirements.txt` (без `^`/`~`).
- Периодически `npm audit`; критичные уязвимости устранять.

## Фронтенд
- `textContent` вместо `innerHTML` для пользовательских данных (защита от XSS).
- Никаких `eval()`, `new Function()`, `setTimeout(string)` с внешними данными.
- Не вставлять непроверенный HTML; для разметки — безопасные шаблоны/санитайзеры.
- Эмодзи в UI — только универсальные (Unicode ≤ 2010 г.), чтобы не рисовались
  «квадратиками» на Windows 10 (см. историю: 🪴/🩺/🥬 заменены).

## Проверка
- Перед пушем: убедиться, что в диффе нет секретов и `.env` не отслеживается git.
- Новый код с сетью/вводом/БД/аутентификацией — проверять по OWASP Top 10.
- Кириллица в командах Bash/PowerShell искажается — искать файлы поиском, а не
  печатать путь вручную; для правки файлов с эмодзи надёжнее Python (UTF-8), не sed/grep.
