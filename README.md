# 🌿 «Зелёный Дом» — сайт про растения, сад и огород

Русскоязычный контентный сайт на **Astro** под **российскую монетизацию**. Быстрый,
бесплатный в хостинге, хорошо индексируется в Яндексе и Google, почти не требует обслуживания.

> ⚠️ **Почему русский, а не английский.** Google AdSense и Amazon сейчас не выплачивают
> деньги на счета в РФ. Поэтому монетизация здесь — **Яндекс РСЯ** + партнёрки российских
> маркетплейсов (Ozon, Wildberries, Яндекс.Маркет): они платят на карту и работают
> с самозанятыми (налог 4–6%, чеки через «Мой налог»).

> ⚠️ **Честно о доходе.** «Пассивный с первого дня» не бывает. Реальный путь: наполнить
> контентом → задеплоить → 3–6+ месяцев на раскрутку в поиске → появляется доход. Дальше
> сайт работает почти сам: раз в неделю глянуть, что не сломалось.

## Как запустить у себя

Нужен интернет (значит — дома). Один раз:

```bash
npm install
```

Локальный просмотр (http://localhost:4321):

```bash
npm run dev
```

Финальная сборка в папку `dist/`:

```bash
npm run build
```

## Как добавлять статьи

### Вариант A — руками (интернет не нужен)

```bash
npm run new -- "Уход за кактусом дома" komnatnye
```

Разделы: `komnatnye`, `sad-ogorod`, `bolezni-vrediteli`, `sovety`.
Адрес создаётся автоматически: русский заголовок → латиница.

### Вариант B — пачкой из списка тем

В `data/topics.json` уже ~31 тема. Сгенерировать 3 черновика:

```bash
npm run generate
```

Без ключа Claude API — **скелеты** для ручного наполнения. С ключом — **готовые статьи**.

### ⚠️ Про черновики

Все сгенерированные статьи создаются с `draft: true` и **не публикуются**, пока ты не
проверишь их и не поставишь `draft: false`. Это защита от штрафов поисковика за мусорный
авто-контент. Вычитывай перед публикацией.

## Настройка — один файл

Открой **`src/site.config.mjs`**:

| Что | Где | Когда заполнять |
|-----|-----|-----------------|
| Название, слоган, описание | `SITE` | сразу |
| Адрес сайта (`url`) | `SITE.url` | после деплоя |
| ID блока Яндекс РСЯ | `MONETIZATION.yandexRtbBlockId` | после одобрения РСЯ |
| Партнёрская метка Ozon | `MONETIZATION.ozonClientId` | после регистрации в партнёрке |
| Номер Яндекс.Метрики | `ANALYTICS.yaMetrikaId` | по желанию |

Пока поля пустые — сайт работает без рекламы и трекинга (так и надо на старте).

## Лунный календарь

Автономное приложение лежит в `public/lunagarden.html` и встроено на странице
`/lunnyj-kalendar` через `<iframe>`. Работает офлайн, без внешних запросов, считает фазы Луны
локально. Чтобы временно скрыть — поставь `hasTool = false` в `src/pages/lunnyj-kalendar.astro`.

## Деплой (бесплатно) — Cloudflare Pages

1. Создай **пустой** репозиторий `zeleny-dom` на GitHub (без README и .gitignore!).
2. Загрузи проект:
   ```bash
   git init && git add . && git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/ТВОЙ_ЛОГИН/zeleny-dom.git
   git push -u origin main
   ```
3. На [dash.cloudflare.com](https://dash.cloudflare.com) → Workers & Pages → Create →
   внизу ссылка **«Хотите развернуть страницы? Начните прямо сейчас!»** → **Pages** →
   *Import an existing Git repository* → выбери `zeleny-dom`.
4. Настройки сборки: **Framework preset: Astro**, Build command `npm run build`, Output `dist`.
5. Deploy. Через минуту сайт на `имя.pages.dev` — впиши адрес в `SITE.url`, закоммить и запушь.

Дальше каждый `git push` пересобирает сайт автоматически.

## Монетизация — пошагово

### Яндекс РСЯ (реклама)

1. Сайт задеплоен, есть контент и служебные страницы (готовы).
2. Подай сайт в Рекламную сеть Яндекса.
3. После модерации создай рекламный блок, возьми его ID (вид `R-A-XXXXXXX-1`) и впиши
   в `MONETIZATION.yandexRtbBlockId`.

### Партнёрки маркетплейсов

1. Зарегистрируйся в партнёрской программе Ozon / Wildberries / Яндекс.Маркет либо в CPA-сети.
2. Получи партнёрскую метку, впиши в `MONETIZATION.ozonClientId`.
3. Партнёрские кнопки на сайте станут реферальными.

## (По желанию) AI-генерация статей

1. Заведи ключ на [console.anthropic.com](https://console.anthropic.com).
2. Скопируй `.env.example` в `.env`, впиши `ANTHROPIC_API_KEY=...`.
3. `npm run generate` — статьи пишутся целиком на русском (всё равно `draft`, вычитывай).

Файл `.env` в git не попадает — ключ остаётся у тебя.
